===================
 Uso como librería
===================

.. sectionauthor:: Antonio Santos Velasco <asantos@viavansi.com>

La implementación de referencia es una librería distribuida en formato *jar* y diseñada para ser
utilizada en implementaciones propias dedicadas al consumo de datos. Como ejemplos de este tipo
podemos señalar las implementaciones existentes para `Apache Cocoon`_ y `Apache Solr`_.

.. _Apache Cocoon: http://cocoon.apache.org
.. _Apache Solr: http://lucene.apache.org/solr

Las implementaciones mencionadas son consumidores de información, que obtienen datos de un servidor
al que se suscriben a través de mecanismos de recolección y modificación y operan con ellos,
posibilitando el manejo de datos provenientes de *iCMS* por parte de servidores no destinados a tal
efecto (en este caso, *Cocoon* y *Solr*). En resumen, las implementaciones de tipo cliente usan los
métodos que están implementados en la librería de referencia del cliente para funcionar como
pasarela entre un sistema existente y un servidor *iCMS*.

Para un listado completo de la API a utilizar, se referencia a la documentación correspondiente al
subproyecto *icms-api*.
