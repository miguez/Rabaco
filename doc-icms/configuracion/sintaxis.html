Sintaxis¶

iCMS utiliza para interpretar los ficheros de configuración los métodos proporcionados por la clase ConfigSlurper, que permiten una sintaxis de configuración más potente que la de un fichero properties convencional.

Las ventajas que tiene utilizar este tipo de configuración son varias:

  • Estructura en árbol de las opciones de configuración, que aportan más claridad y organización.
  • Posibilidad de tipar las opciones de configuración.
  • La posibilidad de definir variables asociadas a un entorno determinado.

Advertencia

Es necesario recordar definir los valores de tipo String utilizando comillas simples o dobles.

A continuación se incluye a modo de ejemplo un fichero de configuración:

/**
* Desarrollado por Viavansi Servicios Avanzados para las Instituciones
* S.L. (VIAVANSI) 2009 Contacto: comercial@viavansi.com
* http://www.viavansi.com
*
* Fichero de configuración del Servidor de ICMS **/

// Configuración del almacenamiento en SDShare
sdshare {
  // Cantidad máxima admitida de fragmentos que conforman un snapshot
  maxSnapshotSize = 10
  // Directorio de almacenamiento
  storageDir = "/opt/icms/data"
}

/** Configuración de ehcache utilizada en el filter.
* Preconfigurada para:
* - Mantener los 1000 objetos más usados en memoria (considerando
    0.5MB por recurso el sistema necesitara 1G)
* - Mantener 8000 objetos en disco
* - Se persiste en disco duro
* - Los objetos vivos permanecen 1 mes en cache.
* - Los objetos muertos permanencen una semana en cache.
* - El intervalo del hilo de expiración es una vez al día **/

cache {
  maxElementsInMemory=1000
  maxElementsOnDisk=8000
  eternal=true
  overflowToDisk=true
  diskPersistent=true
  /** Segundos que permanecen en caché los recursos activos (la caché
  es refrescada en base a notificaciones, por lo que aunque el valor
  sea muy grande los cambios se realizan instantáneamente) **/
  // un mes
  timeToLiveSeconds=2419200
  // una semana
  timeToIdleSeconds=604800
  diskExpiryThreadIntervalSeconds=86400
}


// Parámetros de configuración del servidor Jetty
// http://docs.codehaus.org/display/JETTY/Configuring+Connectors
icms_server {
  // Cola de conexiones
  acceptQueueSize=100
  // Conexiones simultáneas
  acceptors=25
  // Tiempo máximo de espera desde que se inicia la conexión hasta que
  // se recibe el body de la request
  maxIdleTime=1000
}

// Parámetros de configuración del Proxy
// proxy {
// ip=
// port=
// user=
// password=
// }

icms_client {
    // Número de elementos por página.
    default_items_per_page=10
    // Timeout utilizado en las operaciones ICMS.
    timeout=10000000
}


/** Configuración httpclient

* Recomendamos visitar:
* http://hc.apache.org/httpclient-3.x/preference-api.html#HTTP_connection_manager_parameters
*
* Establece el tiempo de espera del socket (SO_TIMEOUT) en
* milisegundos que se empleará durante la ejecución del método.
* Un valor de cero para el tiempo de espera se interpretará como un
* tiempo de espera infinito. **/
http {
    socket_timeout=60000

    // Indica el tiempo de espera hasta que se establezca una conexión.
    // Un valor de cero significa que no existe tiempo de espera.
    connection_timeout=60000

    // Define el número máximo de conexiones permitidas en general.
    // Este valor sólo se aplica al número de conexiones de un caso
    // particular de HttpConnectionManager.
    connection_manager_max_total=100

    // Define el número máximo de conexiones permitidas por host de configuración.
    // Estos valores sólo se aplican a la cantidad de conexiones de un
    // caso particular de HttpConnectionManager.
    // Este parámetro espera un valor de tipo Map. Este valor debería
    // mapear instancias de HostConfiguration a Integers.
    // El valor por defecto puede ser especificado utilizando ANY_HOST_CONFIGURATION.
    connection_manager_max_per_host=50
}

//Configuración WS-* Nivel B
ws {
    username="testUser"
    password="testPass"
}

// buildVersion=${pom.version}.b${BUILD_NUMBER}.r${SVN_REVISION}

Nota

Es posible utilizar también una sintaxis más tradicional en lugar de la estructura en árbol, simplemente utilizando el caracter ‘.’ como separador de niveles.
