Ficheros de configuración¶

La configuración del servicio se realiza a través de ficheros properties localizados en el directorio base del proyecto. Existen tres ficheros de configuración diferentes en la implementación de referencia:

icms-config.properties
Contiene toda la configuración relativa a la operación del servidor y del modo cliente.
icms-providers.properties
Contiene un listado de las colecciones que maneja el sevidor
icms-scripts.properties
Configuración relativa al sistema de eventos.

La carga de las opciones de configuración se realiza de la siguiente manera: los paquetes jar de cliente y servidor incorporan un fichero con valores a utilizar por defecto, que se cargan al arrancar la aplicación (bien como librería o bien de forma independiente). A continuación se examina el directorio base y se cargan las propiedades de configuración que se encuentren allí (si es que las hay), sobreescribiendo los valores que se habían tomado por defecto.

Nota

De esta manera, iCMS es una implementación completamente funcional sin una configuración previa.

Los ficheros de configuración son autoexplicativos, definiendo mediante comentarios la función de cada una de las opciones de configuración. Los ejemplos que se incluyen a continuación no representan las últimas versiones, disponibles en el repositorio de código.

Logging¶

Para activar el sistema de logging de iCMS es necesario que exista un fichero log4j.properties presente en la carpeta data. Se adjunta a modo de ejemplo el fichero de configuración de log4j que se incluye por defecto en el código del servicio iCMS.

# -----------------------------------------------------
# * Desarrollado por Viavansi Servicios Avanzados para las Instituciones S.L. (VIAVANSI) 2007
# * Contacto: comercial@viavansi.com
# * http://www.viavansi.com
# *
# *
# *  Fichero de configuracion de logs de la aplicacion Oficina Virtual
# *  
# * --------------------------------------------------------------------------------


#Configuración para desarrollo
log4j.rootCategory=consolaSimple,fileLogger

# log en fichero
log4j.appender.fileLogger=org.apache.log4j.RollingFileAppender 
log4j.appender.fileLogger.File=logs/icms.log 
log4j.appender.fileLogger.MaxFileSize=5MB 
log4j.appender.fileLogger.MaxBackupIndex=5 
log4j.appender.fileLogger.layout=org.apache.log4j.PatternLayout 
log4j.appender.fileLogger.layout.ConversionPattern=%p %t %c - %m%n
log4j.appender.fileLogger.layout.ConversionPattern=%d{yyyy-MM-dd HH:mm:ss} %-5p %c.%M(%L) - %m%n
log4j.appender.fileLogger.Threshold=DEBUG

# appender appConsola
# Configuracion para utilizar un Appender de Consola
log4j.appender.consolaSimple=org.apache.log4j.ConsoleAppender
# Defino la capa PatternLayout para la consola
log4j.appender.consolaSimple.layout=org.apache.log4j.PatternLayout
log4j.appender.consolaSimple.layout.ConversionPattern=%d{yyyy-MM-dd HH:mm:ss} - %m%n
log4j.appender.consolaSimple.Threshold=INFO

#Configuración de los paquetes con permiso para escribir en consola
log4j.logger=ERROR,consolaSimple
log4j.logger.es.juntadeandalucia.icms.util.ICMSUtil=INFO,consolaSimple

log4j.logger.es.juntadeandalucia.icms.TestUtil=INFO,consolaSimple
log4j.logger.es.juntadeandalucia.icms.TestUtil=DEBUG,fileLogger

#log4j.logger.es.juntadeandalucia.icms=DEBUG,fileLogger
#Nivel de log por paquetes y proyectos
log4j.logger.com.viavansi.framework=INFO,fileLogger
log4j.logger.org.apache.commons=WARN,fileLogger
log4j.logger.httpclient.wire=WARN,fileLogger
log4j.logger.org.mortbay=INFO,fileLogger
log4j.logger.com.hp.hpl.jena=INFO,fileLogger
log4j.logger.org.mortbay.log=WARN,fileLogger
log4j.logger.es.juntadeandalucia.icms.util.ConfigurationUtil=INFO,fileLogger

#Oculta posibles problemas en la ontología de ejemplo.
log4j.logger.com.hp.hpl.jena.ontology.OntDocumentManager=ERROR,fileLogger
log4j.logger.org.apache=INFO,fileLogger
