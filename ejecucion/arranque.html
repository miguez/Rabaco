Uso del servidor¶

El paquete icms-server es una implementación de un servidor iCMS con total funcionalidad, que mediante el uso de protocolos estándares exporta la información que recibe de un sistema productor de contenido a través de interfaces de servicios web de tipo RESTful y SOAP.

Los protocolos de los que se hace uso son:

  • OpenSearch para la descripción del formato de búsqueda de datos.
  • Rdf y Xml para la descripción de contenidos usando la semántica definida por eGov y Dublin Core, entre otras.
  • Atom como formato de respuesta de búsquedas y descripción de colecciones.

A diferencia de la implementación de referencia de cliente, que se utiliza únicamente como librería, el uso habitual que se le va a dar al paquete correspondiente a la implementación del servidor es la ejecución a través de la línea de comandos. No obstante, esto no quiere decir que no sea posible su integración con otras implementaciones a través de llamadas a las clases y métodos escritos en Java.

Como se ha expuesto en el apartado relativo al empaquetado, la distribución de la implementación de referencia servidor cuenta con un paquete que incluye todas las dependencias necesarias para la ejecución del proyecto en el fichero icms-server-1.2.0-All.jar. Este paquete se ha copiado, por otra parte, a un directorio durante el proceso de instalación

Arranque¶

El servidor de referencia iCMS muestra una pequeña página de ayuda al ser ejecutado sin parámetros desde la línea de comandos. Esta página también se muestra si en la ejecución se incluye el parámetro help.

java -jar icms-server-1.2.0-All.jar
configDirectory <arg>
Directorio donde se encuentran los ficheros de configuración.
directoryBase <arg>
Directorio raíz donde se encuentra el servidor iCMS.
help
Este mensaje de ayuda
port <arg>
Permite definir el puerto en el que se encuentra el servidor (por defecto, 9991)
publicUrl <arg>
Url pública
start
Arranca el servidor
version
Información acerca de la versión

Un ejemplo de ejecución desde línea de comandos típica es el siguiente

java -server -jar icms-server-1.1.2-SNAPSHOT-All.jar -publicUrl http://www.example.com
-directoryBase /opt/icms/ -configDirectory /opt/icms/conf -start

Para arrancar el servidor de referencia de iCMS siempre es necesario especificar en la línea de comandos, al menos, el directorio base y la url pública bajo la que se encuentra el servidor.

Nota

El directorio base es uno de los dos parámetros que el servidor necesita recibir para un correcto funcionamiento (ver sección correspondiente). El parámetro directorio base especifica el lugar del sistema de ficheros en que es posible encontrar y escribir los archivos necesarios para la óptima ejecución del servidor iCMS. En dicho directorio se encuentran los ficheros de configuración y el directorio de drivers.

Nota

La url pública se utiliza para definir los enlaces en el servidor, ya que para una correcta operación de iCMS es necesario que los identificadores que se dan a los recursos se ajusten a la dirección desde la que se sirven.

El puerto en el que la aplicación escucha las conexiones se puede definir con el parámetro port, que por defecto toma el valor 9991. Hay que destacar que es posible que la aplicación esté escuchando en dicho puerto y sin embargo tenga otro puerto definido en la url pública, ya que normalmente un balanceador de carga o proxy reverso se encargará de redirigir la petición a la dirección y puerto correspondientes.

El resto de parámetros se pueden especificar no en la línea de comandos, sino a través de los ficheros de configuración de la implementación (véase apartado de configuración).

Script de arranque¶

Para facilitar la ejecución y el uso habitual del servidor iCMS se incluye un script de consola que permite, tras una mínima configuración, el arranque y parada del servidor con los siguientes comandos:

./ctlscript.sh start
./ctlscript.sh stop
./ctlscript.sh restart

El script incluye también un comando clean que permite la limpieza del directorio de ejecución (borrado de ficheros lock)

Para la configuración del fichero, hay que editar las variables de entorno que se encuentran al inicio del mismo.

#!/bin/bash
#
# iCMS:    iCMS webservice daemon
#
# chkconfig: 345 99 50
# description: Run iCMS webservice daemon
#
# processname: icms

# Directorios de instalación
ICMS_HOME="@@ICMS_HOME@@"
ICMS_CONF="$ICMS_HOME/conf"
# Versión del servidor
ICMS_VERSION="@@ICMS_VERSION@@"
ICMS_JAR="$ICMS_HOME/icms-server-$ICMS_VERSION-All.jar"
# Url pública por la cual se accede al servidor
ICMS_URL="@@ICMS_URL@@"
# Puerto en el que escucha el servidor
ICMS_PORT="@@ICMS_PORT@@"
ICMS_PID_DIR="/var/run"
ICMS_PID="$ICMS_PID_DIR/icms.pid"
# Hostname o IP interno para sincronización de caché. Habitualmente, la dirección de red interna de la máquina
ICMS_CACHE_HOSTNAME="@@ICMS_CACHE_HOSTNAME@@"
# Identificador interno que se utilizará para representar el servicio en el log
ICMS_ID="@@ICMS_ID@@"
ICMS_USER="@@ICMS_USER@@"
ICMS_SOLR_WAR="@@ICMS_HOME@@/lib/solr.war"
ICMS_SOLR_HOME="@@ICMS_HOME@@/solr"

RUN_AS_USER="su - $ICMS_USER -c"

JAVA=$(which java)
JAVA_OPTS="-server -Xms16m -Xmx512m -Dlogback.configurationFile=$ICMS_CONF/logback.xml -Dhazelcast.config=$ICMS_CONF/hazelcast.xml -Dicms.cache.hostName=$ICMS_CACHE_HOSTNAME -Dicms.id=$ICMS_ID -Dicms.solr.war=$ICMS_SOLR_WAR -Dsolr.solr.home=$ICMS_SOLR_HOME"
# JAVA_PROFILE_OPTS="-Dcom.sun.management.jmxremote -Dcom.sun.management.jmxremote.port=9000 -Dcom.sun.management.jmxremote.authenticate=false -Dcom.sun.management.jmxremote.ssl=false -agentlib:hprof=heap=dump,file=/tmp/hprof.bin,format=b,depth=10"

# Comentar la siguiente variable si no es necesario el modo debug
# ICMS_DEBUG="-Xrunjdwp:transport=dt_socket,server=y,suspend=n,address=1044 -Xdebug"

# No es necesario modificar a partir de aquí

[ -d $ICMS_HOME ] || (echo "ERROR: no se encontró el directorio $ICMS_HOME" && exit 1)
[ -d $ICMS_CONF ] || (echo "ERROR: no se encontró el directorio $ICMS_CONF" && exit 1)
[ -f $ICMS_JAR ] || (echo "ERROR: no se encontró JAR de iCMS $ICMS_JAR" && exit 1)
[ -d $ICMS_PID_DIR ] || (echo "ERROR: no existe el directorio $ICMS_PID_DIR" && exit 1)

start() {
    echo "Iniciando iCMS..."
    if [ -f "$ICMS_PID" ]; then
        echo "Error: Se ha encontrado un fichero de PID para iCMS. El proceso ya está ejecutándose" 1>&2
        RETVAL=1
    else
		$RUN_AS_USER "$JAVA $ICMS_DEBUG $JAVA_OPTS $JAVA_PROFILE_OPTS -server -jar $ICMS_JAR -publicUrl $ICMS_URL -port $ICMS_PORT -directoryBase $ICMS_HOME -configDirectory $ICMS_CONF -start" &
        RETVAL=$?
        echo $! > $ICMS_PID
        if [ $RETVAL -gt 0 ]; then
            $RUN_AS_USER "rm -f $ICMS_PID" > /dev/null 2> /dev/null
        fi
    fi
    return $RETVAL
}

stop() {
    echo "Deteniendo iCMS..."
    if [ -f "$ICMS_PID" ]; then
        RUNNING_PID=$(cat $ICMS_PID)
        kill $RUNNING_PID
        RETVAL=$?
        clean
    else
        echo "Error: No se encuentra el fichero de PID $ICMS_PID" 1>&2
        RETVAL=1
    fi
    return $RETVAL
}

clean() {
    if [ -f "$ICMS_PID" ]; then
        rm "$ICMS_PID"
    fi
    echo "iCMS dir cleanup performed" }


case "$1" in
    start)
        start
        ;;
    stop)
        stop
        ;;
    restart)
        stop
        sleep 5
        start
        ;;
    clean)
        clean
        ;;
    *)
        echo "Usage: $0 {start|stop|restart|clean}"
        ;;
esac
exit $RETVAL

Configuración como servicio en sistemas Red Hat o compatibles¶

El script de instalación expuesto en el apartado anterior es compatible con sistemas Red Hat o compatibles. Simplemente es necesario copiarlo a la carpeta /etc/rc.d/init.d para poder utilizar el comando service del sistema.
